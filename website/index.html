Protocol Design

Six primitives. Zero ambiguity.

ARP defines three server primitives (robot to LLM) and three client primitives (LLM to robot) over JSON-RPC 2.0.

Server Primitives — Robot → LLM

⚙

Physical Tools

Callable robot actions with parameter schemas, safety metadata, preconditions, effects, and a full execution state machine.

arp.callTool
📡

Physical Context

Streaming sensor and state data — odometry, camera feeds, joint states, lidar — with coordinate frames and configurable rates.

arp.subscribeContext
🛡

Safety Constraints

Non-overridable rules enforced at the protocol level. Velocity limits, workspace bounds, force limits, collision zones, emergency stop.

arp.listConstraints

Client Primitives — LLM → Robot

📝

Planning

Server requests the LLM to generate or revise an action plan given the current state, available tools, and active constraints.

arp.requestPlan
📌

Workspace

Define physical and logical boundaries the robot operates within, including known objects and their types.

arp.setWorkspace
✅

Confirmation

Server requests human approval before executing dangerous actions. Auto-rejects on timeout. Safety level determines when required.

arp.requestConfirmation
How It Works

Architecture

ARP sits between the LLM agent and the robot hardware. The server runs on or near the robot; the client runs alongside your agent.

Comparison

MCP vs ARP

ARP is inspired by MCP but designed specifically for the unique challenges of physical robot control.

MCP ARP
Domain Digital tools & APIs Physical robots
Safety Application-level Protocol-level enforcement
Real-time Request/response Streaming context + progress
E-Stop N/A Built-in emergency stop
Confirmation N/A Human-in-the-loop primitive
Coordinate Frames N/A Native support
Transport stdio / SSE WebSocket (JSON-RPC 2.0)
Get Started

Quick Start

Install the Python SDK and connect your first LLM agent to a robot in minutes.

  terminal 
# Install the SDK
pip install arp-sdk

# Or install from source
git clone https://github.com/mariagorskikh/robotcontextprotocol.git
cd robotcontextprotocol/sdk/python
pip install -e .
  server.py — Define your robot 
from arp_sdk import ARPServer, SafetyMetadata, SafetyLevel

server = ARPServer(name="my-robot", robot_model="Robot Arm v2")

@server.tool(
    description="Move arm to [x, y, z]",
    safety=SafetyMetadata(level=SafetyLevel.NORMAL),
    parameters={"target": {"type": "array"}},
)
async def move_to(target: list[float]) -> dict:
    # Your robot control code here
    return {"reached": target}

asyncio.run(server.run())  # ws://localhost:8765
  agent.py — Control from your LLM agent 
from arp_sdk import ARPClient

client = ARPClient("ws://localhost:8765")
await client.connect()
await client.initialize()

tools = await client.list_tools()      # Discover capabilities
result = await client.call_tool(         # Execute actions
    "move_to", target=[1.0, 0.5, 0.0]
)
print(result.state)  # "completed"
